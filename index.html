도움말

1. 필요한 파일 및 로드

다음 Excel 파일을 앱과 같은 폴더에 두면 페이지 로드 시 자동으로 읽습니다.

  • kb023_information.xlsx — 계약자 목록. 첫 행은 헤더, 그 다음 행부터 데이터입니다. 계약자명, 주민번호, 배정일자·종료일자, 문자전송, DB종류, 연락처 등이 있으면 화면·필터에 사용됩니다.
  • kb023_insurance.xlsx — 보험 데이터. 계약자명·주민번호로 information과 매칭됩니다. 컬럼명 "보험종류"는 화면에서 "상품종류"로 표시됩니다. 월보험료·납입보험료·총보험료·해지환급금·대출금은 금액(천 단위 콤마)으로 표시됩니다.
  • kb023_accident.xlsx — 사고/사전교섭 데이터. 계약자명·주민번호로 필터됩니다. accident 모드(입력/수정/삭제)에서 완료 시 /api/accident로 저장할 수 있습니다.
  • kb023_counsel.xlsx — 과거 상담 기록. 계약자명·주민번호로 필터되어 counsel 영역에 문/답 형식으로 표시됩니다.
  • kb023_products.xlsx — 상품 목록. 보험사·상품종류·상품명 등이 있으면 DB고객 세부내용의 상품종류·상품명 드롭다운에 사용됩니다.

브라우저 개발자 도구(F12) → 콘솔에서 각 파일 로드 시 컬럼 목록과 "컬럼 없음(매칭안됨)" 경고를 확인할 수 있습니다. 시트 첫 행(헤더)과 코드에서 기대하는 컬럼명이 맞는지 확인하세요.

2. information 테이블

  • 페이지를 열면 information 시트가 왼쪽 상단 테이블에 표시됩니다.
  • 행을 클릭하면 해당 계약자의 DB고객 세부내용, insurance, accident, counsel이 오른쪽/아래 영역에 표시됩니다. 동의/부동의는 information의 "문자전송" 컬럼 값으로 채워집니다.

3. DB고객 세부내용

  • information에서 행을 선택한 뒤에만 드롭다운(조회/입력/수정/삭제/출력)과 완료 버튼이 활성화됩니다. 수정·삭제·출력 옵션은 행 선택 후에만 선택 가능합니다.
  • 조회: 선택한 행의 세부내용·보험 데이터를 합쳐서 폼에 채웁니다. 동의/부동의는 information의 문자전송 값으로 설정됩니다.
  • 입력: 폼을 비운 뒤 새로 입력하고 완료 시 저장합니다(백엔드 연동 시).
  • 수정/삭제: 선택한 행 데이터를 수정하거나 삭제한 뒤 완료로 반영합니다.
  • 출력: 출력을 선택한 뒤 완료를 누르면 선택한 행 기준으로 DB고객 세부내용·kb023_insurance·kb023_accident·kb023_counsel을 A4로 인쇄합니다. 페이지의 머리글/바닥글을 제거하려면 인쇄 대화상자에서 머리글 및 바닥글 옵션을 해제하세요.
  • 배정일자·종료일자·납입만기 입력란에 숫자 6자리(yymmdd) 또는 8자리(yyyymmdd)를 입력한 뒤 포커스를 벗어나면 "yy/mm/dd" 형식으로 자동 변환됩니다.

4. insurance 테이블

  • 선택한 계약자의 보험 데이터가 표시됩니다. 계약자명·주민번호 컬럼은 테이블에서 숨겨집니다.
  • 행을 클릭하면 DB고객 세부내용의 "계약상세" 영역(상품종류·상품명·증권번호·납입만기·보험료 등)만 해당 보험 행 데이터로 갱신됩니다.
  • 월보험료·납입보험료·총보험료·해지환급금·대출금은 금액으로 포맷(천 단위 콤마, 우측 정렬)됩니다. 날짜는 yy/mm/dd로 표시됩니다.

5. accident 테이블

  • 계약자를 선택한 뒤 드롭다운(입력/수정/삭제)과 완료 버튼이 활성화됩니다.
  • 입력: 마지막 행에 사고일자·사고내용 등을 입력합니다. 사고일자란에 6자리 또는 8자리 숫자만 입력 후 포커스를 벗기면 "yy/mm/dd"로 변환됩니다. 완료를 누르면 /api/accident로 전송되어 xlsx에 저장되고 테이블이 다시 그려집니다.
  • 수정: 수정할 행을 클릭하면 해당 행이 편집 가능한 입력란으로 바뀝니다. 사고일자·사고내용 등을 수정한 뒤 완료를 누르면 화면 데이터가 갱신되고 테이블이 재출력됩니다.
  • 삭제: 삭제할 행을 클릭하면 행이 강조 표시됩니다. 완료를 누르면 "해당 행을 삭제합니다" 메시지가 뜨고, 확인 후 테이블이 다시 그려집니다.

6. counsel(상담)

  • 과거 상담: 선택한 계약자의 counsel 데이터가 문/답 말풍선으로 표시됩니다. 말풍선 옆에 "yy/mm/dd (요일) HH:MM" 형식으로 표시됩니다.
  • 오늘 상담: 문(가입자)/답(상담자) 드롭다운을 선택한 뒤 메시지를 입력하고 전송합니다. 문으로 전송하면 드롭다운이 답으로, 답으로 전송하면 문으로 자동 변경됩니다. 전송 시 /api/counsel로 저장됩니다.
  • 말풍선 너비는 "오늘 상담 메시지 입력" 입력 필드 너비와 맞춰지며, 배경색은 50% 투명으로 표시됩니다.

7. 배정자료 모달

  • 헤더의 "배정자료" 버튼을 누르면 information·insurance 데이터를 배정일자·계약자 순으로 모달에 표시합니다. seq/배정일자/문자전송/DB종류/계약자명/주민번호/연락처/증권번호/납입만기가 한 줄로 표시됩니다.
  • 모달은 헤더를 드래그하거나 방향키로 이동할 수 있습니다. 원위치 버튼을 누르면 화면 중앙으로 돌아갑니다.
  • 모달 영역 밖(배경)을 클릭하면 모달이 닫힙니다. 테이블 행을 클릭하면 모달이 닫히고 해당 information 행이 선택된 상태로 메인 화면에 표시됩니다.

8. 푸터

  • 작업종료 버튼을 누르면 "작업을 종료하시겠습니까?" 확인 후 창을 닫습니다. (일반 탭에서는 브라우저 정책에 따라 닫히지 않을 수 있습니다.)

9. 주의사항

  • Excel 파일은 앱과 같은 경로에 두어야 하며, 시트의 첫 행은 반드시 헤더(컬럼명)로 사용됩니다. 컬럼명이 코드와 다르면 해당 기능이 동작하지 않을 수 있으므로, 도움말의 "필요한 파일 및 로드"와 콘솔 로그를 참고해 맞추세요.
  • 주민번호·연락처 등 개인정보가 화면에 표시됩니다. 사용 환경에 맞게 접근 제한, 화면 보호 등 보안을 유지하세요.
  • DB고객 세부내용의 입력/수정/삭제, accident 입력, counsel 전송 등은 서버 API(/api/accident, /api/counsel 등)와 연동됩니다. Python 서버(app.py)를 실행한 상태에서 사용해야 저장이 반영됩니다. 저장 실패 시 알림이 뜨며 데이터가 반영되지 않을 수 있습니다.
